Credit Card Fraud Detection
using Classical Machine Learning

Can machine learning models reliably identify fraudulent credit card transactions from anonymized behavioral data — and which classical algorithm performs best under severe class imbalance?

Problem Statement

Credit card fraud causes billions in financial losses globally each year. In India alone, over ₹2,600 crore was lost to digital payment fraud in 2023. The core challenge is detecting fraud in real-time from millions of transactions, where fraudulent cases account for only 0.17% of all activity — making it extremely difficult for models to learn fraud patterns without special handling.

Literature Review / Market Research

Existing industry solutions such as Visa's Advanced Authorization and Mastercard's Decision Intelligence use deep neural networks and graph ML, processing 500+ features per transaction with 99.9%+ accuracy. Academic research extensively uses the ULB Credit Card dataset (Kaggle) as a benchmark — with studies comparing Logistic Regression, Random Forest, XGBoost, and neural networks. SMOTE-based oversampling has been established as the standard approach to handle class imbalance in fraud detection tasks.

Research Gap / Innovation

Most student-level implementations use a single model without systematic comparison. This project addresses that gap by building, evaluating, and comparing two distinct classical ML models — Random Forest and XGBoost — under identical preprocessing and SMOTE conditions, providing a clear, evidence-based model selection decision backed by multiple evaluation metrics.

System Methodology

Dataset / Input

Source
Kaggle — ULB Credit Card Fraud Detection Dataset (Université Libre de Bruxelles + Worldline)
Size
284,807 transactions · 31 columns · 2 days of real European cardholder data (Sept 2013)
Features
Time, Amount, V1–V28 (PCA-anonymized behavioral features), Class (0 = Genuine, 1 = Fraud)
Preprocessing
StandardScaler on Amount & Time · SMOTE to fix 0.17% class imbalance · 80/20 stratified train-test split

Model / Architecture

1
Data Loading & Exploration Load CSV → verify shape (284,807 × 31) → visualize class distribution
2
Preprocessing Scale Amount & Time using StandardScaler · separate features (X) and target (y)
3
SMOTE Balancing Applied only on training data · balanced fraud from 394 → 227,451 synthetic samples
4
Model Training Random Forest (100 trees, max_depth=10) · XGBoost (100 trees, lr=0.1, max_depth=6)
5
Evaluation & Comparison Precision · Recall · F1-Score · ROC-AUC · Confusion Matrix · Feature Importance
